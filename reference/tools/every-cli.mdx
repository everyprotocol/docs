Command line interface for the Every Protocol

## Install

```sh-session
$ npm install -g @everyprotocol/every-cli
```

## Usage

```sh-session
$ every COMMAND
running command...
$ every (--version)
@everyprotocol/every-cli/0.0.3 darwin-arm64 node-v22.6.0
$ every --help [COMMAND]
USAGE
  $ every COMMAND
...
```

### `every autocomplete [SHELL]`

Display autocomplete installation instructions.

```
USAGE
  $ every autocomplete [SHELL] [-r]

ARGUMENTS
  SHELL  (zsh|bash|powershell) Shell type

FLAGS
  -r, --refresh-cache  Refresh cache (ignores displaying instructions)

DESCRIPTION
  Display autocomplete installation instructions.

EXAMPLES
  $ every autocomplete

  $ every autocomplete bash

  $ every autocomplete zsh

  $ every autocomplete powershell

  $ every autocomplete --refresh-cache
```

_See code: [@oclif/plugin-autocomplete](https://github.com/oclif/plugin-autocomplete/blob/v3.2.26/src/commands/autocomplete/index.ts)_

### `every balance transfer WHO AMOUNT`

Send tokens from your account to another address

```
USAGE
  $ every balance transfer WHO AMOUNT -a <value> [-c <value>] [-u <value>] [-k
    <value>] [-p <value>] [--password-file <value>]

ARGUMENTS
  WHO     The recipient address
  AMOUNT  The amount of token to transfer

CONTEXT FLAGS
  -a, --account=<value>        (required) Name of the keystore
  -c, --conf=<value>           Specify config file
  -k, --keystore-dir=<value>   Keystore directory
  -p, --password=<value>       Password to unlock the keystore
  -u, --universe=<value>       Universe name in the config file
      --password-file=<value>  Password file to unlock the keystore

DESCRIPTION
  Send tokens from your account to another address
```

### `every kind register CODE DATA [SHAPE] [RELS]`

Registers a kind at the kind registry

```
USAGE
  $ every kind register CODE DATA [SHAPE] [RELS] -a <value> [--json] [-c
    <value>] [-u <value>] [-k <value>] [-p <value>] [--password-file <value>]

ARGUMENTS
  CODE   The material hash of the kind contract
  DATA   The material hash of the kind data
  SHAPE  [default: ] Element types
  RELS   [default: ] Supported relation ids

CONTEXT FLAGS
  -a, --account=<value>        (required) Name of the keystore
  -c, --conf=<value>           Specify config file
  -k, --keystore-dir=<value>   Keystore directory
  -p, --password=<value>       Password to unlock the keystore
  -u, --universe=<value>       Universe name in the config file
      --password-file=<value>  Password file to unlock the keystore

GLOBAL FLAGS
  --json  Format output as json.

DESCRIPTION
  Registers a kind at the kind registry
```
