# Objects

## 1. Tokens and Token Contracts

The OpenZeppelin documentation[^note] explains the distinction:

> *Token contracts are smart contracts that manage token balances and operations, while tokens themselves are the digital assets represented by those balances.*

Traditional tokens maintain simple mappings like:
```plain
// ERC-20
address => balance

// ERC-721/ERC-1155  
id => (owner, metadata)
```

## 2. Token Records in Standards

### ERC-20
- Single balance per account
- Fungible tokens
- Simple transfer operations

### ERC-721
- Unique token IDs
- Non-fungible tokens (NFTs)
- Ownership tracking

### ERC-1155
- Both fungible and non-fungible
- Batch operations
- More complex state management

## 3. Object Token Records

Objects extend traditional tokens with:
```plain
id => (owner, state)  // Set contract
id => pos             // OOPS (Object Operating and Positioning System)
```

Key differences:
- Structured state via elements
- Position tracking
- Built-in relations

## 4. Core Components

### Meta
```solidity
struct Meta {
    uint32 flags;
    uint32 rev;
    uint32 kindRev;
    uint32 setRev;
    uint64 kind;
    uint64 set;
}
```
- Tracks object type, revisions and status

### Elements
- **Matter**: Digital content
- **Value**: Fungible tokens  
- **Unique**: NFTs
- **Info**: Arbitrary data

### Connections
- Relations between objects
- Position in virtual space
- Cross-chain capabilities

### Facets
- Set contracts (creation logic)
- Kind contracts (behavior logic)

## 5. Basic Operations

| Operation | Description | Fields Affected |
|-----------|-------------|-----------------|
| transfer  | Change owner | owner |
| update    | Modify elements | meta.rev, elems |
| upgrade   | Change kind/set | meta.rev, meta.kindRev, meta.setRev |
| touch     | Increment revision | meta.rev |

## 6. Inter-Operations

| Operation | Description | Fields Affected |
|-----------|-------------|-----------------|
| relate    | Connect objects | meta.rev, pos, owner |
| unrelate  | Disconnect objects | meta.rev, pos, owner |
| move      | Change position | pos |
| enter     | Join a world | pos, owner |
| jump      | Cross-chain transfer | pos, owner |

## Compatibility

Objects maintain backward compatibility with:
- ERC-721
- ERC-1155  
- Other NFT standards

Enabling integration with existing:
- Wallets
- Marketplaces
- Block explorers
